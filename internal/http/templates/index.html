Protocol endpoints

  • /.well-known/openid-configuration — OIDC discovery document
  • /.well-known/jwks.json — public signing keys
  • /authorize — start an OAuth auth-code flow (needs query params; use make oauth-url)
  • /token — exchange an auth code for tokens (POST, called by the client's backend)
  • /userinfo — user claims for a valid access token (GET with Authorization: Bearer)

Admin / dev

  • /healthz — liveness + Postgres ping
  • /login and /consent are step-throughs inside the OAuth flow — don't hit them directly

Getting a working /authorize URL

make oauth-url prints a paste-ready URL with a fresh PKCE challenge. Or drive the full flow via make dev-flow.
